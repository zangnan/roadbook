## 功能需求与实现记录

### 1. inputtips API 修复

**问题**：输入地点时搜索下拉框不显示

**原因**：`handleInputChange` 函数传递的是 `this.value`（字符串），而 `searchAddress` 需要整个输入框元素来定位下拉框位置

**修复**：
- `oninput="handleInputChange('${loc.id}', this.value)"` 改为 `oninput="handleInputChange('${loc.id}', this)"`
- 函数参数从 `value` 改为 `input`，内部取 `input.value`

### 2. UI 浮层改造

**需求**：
1. 左侧"地点列表"及右侧"路线概览"改为浮层形式，可收起展开
2. 添加交换起终点按钮
3. 地点列表紧凑化，一行显示

**实现**：

- 地点列表改为紧凑模式，图标+输入框+删除按钮一行显示

---

### 3. 高德 API 多路线方案

**需求**：仿照高德地图，显示推荐方案、躲避拥堵、避免收费等选项

**分析**：高德 API 的 `strategy` 参数：
- 0：速度优先
- 1：费用优先
- 2：常规最快
- 32：高德推荐（默认）
- 33：躲避拥堵
- 34：高速优先
- 35：不走高速
- 36：少收费
- 37：大路优先
- 38：速度最快
- 39-45：组合策略

**实现方案**：
- 推荐方案 = 32 + 0 + 2（调用3次API）
- 最短路线 = 0 + 37 + 4
- 避免收费 = 1 + 36 + 41

---

### 4. 出行方式全局化

**需求**：
1. 出行方式不是每段单独选择，而是全局设置
2. 出行方式放在面板页头
3. 出行策略仅驾车时显示，也作为全局设置

**实现**：
- `currentTransportMode` 全局变量存储当前交通方式
- `currentDrivingPlan` 全局变量存储当前驾车策略
- 交通方式按钮组在左侧面板页头
- 出行策略按钮在右侧面板（开始规划后显示）

### 7. 方案卡片高亮修复

**问题**：点击方案卡片后高亮不生效，被 `renderRouteResult` 重新渲染清除

**修复**：
- 新增全局变量 `selectedRouteOption` 记录选中索引
- `switchRouteOption()` 只更新索引和路线数据，不操作 DOM
- `showRouteSummaryFromResult()` 渲染时根据索引应用 `active` 样式
- 开始规划/清除时重置索引为 0
